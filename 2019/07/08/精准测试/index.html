痛点

  1. 迭代更新快
  2. 多分支代码何如到主干代码
  3. 代码更新影响的功能无法感知
  4. 无法精准监控代码质量

星云测试

  • git diff针对版本号之间进行差异化分析
    • 过滤相关文件 test 公共库等等文件
    • 记录当前代码分支号
    • 针对新版本号和旧版本号 文件中航变化的明细入库
  • git diff
    • -代表源文件 +表示目标文件 空格开头的是源文件和目标文件都出现的行 差异按照差异小姐的第一行都是定位于巨，有@@开头 @@结尾

  1. 执行git diff 目标版本 已知最新正确版本 获取到结果放置到文件中

  2. 根据diff –git a/* b/* 进行分块 获取到所有改动的源文件与目标文件和内容

  3. 根据文件类型、地址等等筛选出关心的程序模块，解析出源文件改动的代码地方、和新添加的代码地方

  4. 源文件 名称 目录 修改行数 | 新文件 名字 目录 修改行数

  5. dump到jacoco的当前覆盖率

  6. 根据diff出的diff行内容，以及jacoco覆盖率情况，在源码中涂色，整理出diff。html

    • table 行号 内容
    • 区分line line_new line_old background展示 可以学习github的方式！
