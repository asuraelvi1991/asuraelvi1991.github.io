  3. 登陆到统一界面，群聊
  4. 统一界面，@特定的人，只有特定人可以接收消息

用户连接到sever，是一个个socket，多用户连接则需要维护一个socket容器。
List,

问题1，如何区别不同用户建立的链接

  1. 创建channel，list用线程安全的，每次新建链接就添加到list中
  2. channel对象中存有名字，在初始化channel时就传入
  3. 特殊的文字格式给@xxx：内容，私聊
  4. release在各种异常中释放，将所有内容释放，并把socket从list中移除

问题2，实时交流，因此群聊的输入流和输出流需要分开多线程去转发，不阻塞流程。

  1. 实现runnable接口，起线程去处理请求
  2. client端将接受、和请求分开，因为可以同时接受并发送消息
